# 项目开发总结报告

## 1 引言

### 1.1 编写目的

本文档用于总结“今天吃什么（What-2-Eat）”项目当前阶段的软件开发结果，说明项目产品、功能性能、流程、进度、费用、质量和开发经验，供项目组成员、课程指导教师和验收人员阅读与评审。

### 1.2 背景

a．本项目名称为“今天吃什么（What-2-Eat）课程项目”，所开发的软件系统名称为“今天吃什么（What-2-Eat）智能餐饮推荐系统”。

b．结合当前仓库和现有文档，可整理出背景如下：

1. 任务提出者：课程项目教学任务提出方与项目指导教师。
2. 开发者：What-2-Eat 项目开发小组成员。
3. 用户：项目组成员、课程演示人员、课程验收试用人员。
4. 安装此软件的计算中心：当前版本主要安装于项目组成员本地开发与演示计算机。仓库中未单独记录正式计算中心名称，提交定稿时可替换为具体实验室或机房名称。

### 2.3 基本流程

本程序系统实际的基本处理流程如下：

```text
用户输入推荐条件
    ↓
前端页面提交请求
    ↓
后端路由接收请求
    ↓
意图识别模块解析自然语言
    ↓
高德地图接口获取候选餐馆
    ↓
数据清洗与标准化
    ↓
推荐引擎评分排序
    ↓
解释模块生成推荐理由
    ↓
返回 JSON 响应给前端
    ↓
前端展示推荐结果
    ↓
用户可提交反馈并记录到数据库
```

## 3 开发工作评价

### 3.1 对生产效率的评价

当前仓库未保留正式工时台账，因此无法像工业项目一样精确给出“每人月生产行数”的严格统计值。本节按当前成果规模进行阶段性评价。

a．程序平均生产效率：

1. 当前后端、前端、数据库脚本、测试和文档已经形成完整课程项目原型。
2. 从开发结果看，项目组在较短时间内完成了后端 API、推荐主流程、前端原型、MySQL 持久化和测试基础，说明阶段生产效率较高。

b．文件平均生产效率：

1. 当前仓库文档体系较完整，已形成接口文档、设计文档、数据库文档、解释系统文档和工作总结。
2. 文件产出与代码实现保持了较高同步性，有助于后续联调与答辩准备。

与原订计划相比，本项目在“核心原型形成速度”方面表现较好，尤其是主链路落地速度高于很多仅停留在方案阶段的课程项目。

### 3.2 对产品质量的评价

从当前产品质量看，可做如下评价：

1. 项目已具备后端测试文件、前端单元测试文件和端到端测试脚本。
2. 候选餐馆获取、数据清洗、推荐打分和解释逻辑均已有测试覆盖基础。
3. 产品主链路可运行，接口文档齐全，说明可用性较好。

当前不足如下：

1. 尚未形成完整缺陷台账，无法精确计算“每千条指令错误数”。
2. 真实用户系统、生产级权限控制和分布式能力仍未完成，因此产品质量评价主要限于课程原型层面。

总体评价如下：

1. 作为课程项目原型，产品质量较好。
2. 作为生产系统，仍需在安全性、鲁棒性和规模化部署方面继续加强。

### 3.3 对技术方法的评价

开发中使用的主要技术、方法、工具和手段评价如下：

1. `FastAPI + Pydantic` 适合快速构建结构清晰的后端接口系统。
2. `Vue 3 + Vite` 适合课程项目阶段快速搭建前端交互原型。
3. `SQLAlchemy + MySQL` 便于在课程项目环境下实现更接近真实部署形态的数据库落地。
4. “规则推荐 + AI 解释/意图识别”的技术路线兼顾了可控性与扩展性。
5. 先构建主链路、再逐步增加解释、历史、反馈和扩展特性的方式，有利于降低开发风险。

总体来看，当前采用的技术方法适合课程项目原型开发，选型合理，开发效率较高。

### 3.4 出错原因的分析

开发中可能出现或已经体现出的主要错误原因如下：

1. 前后端字段命名在迭代中多次演进，容易导致联调不一致。
2. 外部地图接口和大模型接口具有不稳定性，可能导致空结果或超时。
3. 用户系统尚未后端化，导致前端模拟状态与后端真实数据之间存在差异。
4. 设计文档中的部分理想能力与当前实现之间存在阶段差距，容易造成预期偏差。
5. 课程项目采用并行协作开发，若文档同步不及时，容易产生模块理解偏差。

## 4 经验与教训

从本次开发工作中得到的主要经验与教训如下：

1. 先打通“输入、处理、输出”的主链路，再逐步增强细节，是课程项目中最有效的推进方式。
2. 接口文档和设计文档必须尽早维护，否则多人协作时容易出现字段和逻辑偏差。
3. 对外部依赖必须设计降级策略，尤其是地图接口和大模型接口。
4. 课程项目中的“可运行原型”比“只停留在纸面上的大而全方案”更有价值。
5. 用户系统、数据持久化和解释系统应尽早统一数据结构，避免后期返工。
6. 若后续继续开发，建议优先完成真实用户系统、前后端统一字段、历史闭环和缓存优化。

总体而言，本项目已经形成了一个可启动、可联调、可演示、可继续扩展的智能餐饮推荐系统原型，为后续完善和课程答辩奠定了较好的基础。
