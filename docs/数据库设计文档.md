## 数据库设计目标

本系统数据库主要用于支持以下核心业务：

- 用户基础信息管理
- 用户偏好与画像存储
- 周边餐馆信息管理
- 查询记录与推荐记录保存
- 用户反馈闭环优化
- 历史行为分析与推荐优化

数据库设计遵循以下原则：

- 结构清晰，满足系统核心业务流程
- 支持推荐闭环与后续算法扩展
- 保证一定的查询性能与可维护性
- 支持后期个性化推荐与行为分析
- 适合课程项目开发规模与部署需求

本项目采用 MySQL 作为主要关系型数据库。

------

# 1. 核心实体关系

系统主要包含以下核心实体：

| 实体                  | 作用                                     |
| --------------------- | ---------------------------------------- |
| user                  | 存储用户标识                             |
| tag                   | 核心标签体系（菜系、口味、场景等）       |
| user_tag_preference   | 用户长期的个性化标签偏好权重             |
| restaurant            | 存储餐馆基础信息缓存                     |
| restaurant_tag        | 餐馆与标签的打标关联关系                 |
| user_query            | 存储用户每次查询请求的具体参数           |
| recommendation        | 存储推荐给用户的具体餐厅及其评分、理由   |
| feedback              | 存储用户对推荐结果的真实反馈（评分、选择）|
| interaction           | 用户行为日志（点击、LIKE、DISLIKE等）     |

# 2. 数据库表设计

------

# （1）标签体系表 tag

系统核心标签库，支持父子级嵌套。

```sql
CREATE TABLE tag (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL UNIQUE,
    type        TEXT NOT NULL,           -- cuisine / taste / type / scene / feature
    description TEXT,
    parent_id   INTEGER REFERENCES tag(id)
);
```

------

# （2）用户表 user

仅存储基础标识，详细画像由偏好表支持。

```sql
CREATE TABLE user (
    id          TEXT PRIMARY KEY,
    created_at  TEXT
);
```

------

# （3）用户偏好表 user_tag_preference

存储用户对各标签的偏好分值，用于推荐排序。

```sql
CREATE TABLE user_tag_preference (
    user_id     TEXT NOT NULL,
    tag_id      INTEGER NOT NULL,
    preference  REAL DEFAULT 0.5,
    updated_at  TEXT,
    PRIMARY KEY (user_id, tag_id)
);
```

------

# （4）餐馆缓存表 restaurant

保存通过地图服务获取到的餐馆快照。

```sql
CREATE TABLE restaurant (
    id          TEXT PRIMARY KEY,        -- 高德 POI ID
    name        TEXT NOT NULL,
    category    TEXT,                    -- 高德原始分类
    address     TEXT,
    latitude    REAL,
    longitude   REAL,
    rating      REAL DEFAULT 0,
    avg_price   REAL DEFAULT 0,
    updated_at  TEXT
);
```

------

# （5）餐馆-标签关联表 restaurant_tag

```sql
CREATE TABLE restaurant_tag (
    restaurant_id TEXT NOT NULL,
    tag_id        INTEGER NOT NULL,
    weight        REAL DEFAULT 1.0,
    PRIMARY KEY (restaurant_id, tag_id)
);
```

------

# （6）查询记录表 user_query

记录用户每次请求推荐时的上下文环境。

```sql
CREATE TABLE user_query (
    id          TEXT PRIMARY KEY,        -- UUID
    user_id     TEXT,
    longitude   REAL,
    latitude    REAL,
    radius      INTEGER,
    budget_min  REAL,
    budget_max  REAL,
    taste       TEXT,
    query_text  TEXT,
    created_at  TEXT
);
```

------

# （7）推荐结果表 recommendation

记录系统生成的推荐项及具体的评分明细与解释。

```sql
CREATE TABLE recommendation (
    id              TEXT PRIMARY KEY,    -- UUID
    query_id        TEXT REFERENCES user_query(id),
    restaurant_id   TEXT,
    restaurant_name TEXT,
    rank            INTEGER,
    final_score     REAL,
    score_distance  REAL,
    score_price     REAL,
    score_rating    REAL,
    score_tag       REAL,
    matched_tags    TEXT,               -- JSON字符串
    reason_hint     TEXT,               -- JSON字符串
    explain_json    TEXT,               -- 完整结构化解释JSON
    created_at      TEXT
);
```

------

# （8）用户反馈表 feedback

记录用户对推荐结果的评价，作为偏好更新的基准。

```sql
CREATE TABLE feedback (
    id                TEXT PRIMARY KEY, -- UUID
    user_id           TEXT,
    recommendation_id TEXT REFERENCES recommendation(id),
    restaurant_id     TEXT,
    rating            INTEGER,
    chosen            INTEGER DEFAULT 0,
    created_at        TEXT
);
```

------

# （9）行为交互表 interaction

记录精细化的用户行为，用于长短期画像建模。

```sql
CREATE TABLE interaction (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id       TEXT NOT NULL,
    restaurant_id TEXT NOT NULL,
    action_type   TEXT,                  -- click / LIKE / DISLIKE
    score         REAL,
    timestamp     TEXT
);
```

# 4. 数据库优化设计

## （1）存储选型
采用 SQLite 替代原定的 MySQL，核心考量是课程项目的一键部署与零配置运行特性。

## （2）反规范化设计
在 `recommendation` 表中冗余存储 `restaurant_name`，避免生成解释结果时的高频联表查询。

## （3）JSON 字段说明
SQLite 不支持原生 JSON 类型（使用 TEXT 存储），系统在应用层（Pydantic）解析：
- `matched_tags`: 匹配成功的标签列表，如 `["火锅", "辣"]`
- `reason_hint`: 推荐理由摘要，如 `["口味匹配", "距离近"]`
- `explain_json`: 包含 `hello_voice` 和 `match_details` 的完整前端呈现对象内容
