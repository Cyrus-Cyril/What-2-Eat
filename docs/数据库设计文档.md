# 1. 文档概述

## 1.1 文档目的

本文档用于说明“今天吃什么”智能推荐系统的数据库结构设计。

数据库主要用于实现：

* 餐厅信息缓存
* 标签化推荐体系
* 用户偏好建模
* 推荐结果存储
* 用户行为分析
* 推荐反馈闭环

系统数据库设计支持：

```text id="p4c0wk"
用户查询
→ 并行推荐
→ 推荐排序
→ 用户反馈
→ 偏好更新
```

完整推荐链路。

# 4. 数据表设计

# 4.1 标签表（tag）

用于管理餐厅标签体系。

---

## 表结构

```sql id="nhq0k3"
CREATE TABLE tag (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL UNIQUE,
    type        TEXT NOT NULL,
    description TEXT,
    parent_id   INTEGER REFERENCES tag(id)
);
```

---

## 字段说明

| 字段          | 类型      | 说明    |
| ----------- | ------- | ----- |
| id          | INTEGER | 标签ID  |
| name        | TEXT    | 标签名称  |
| type        | TEXT    | 标签类型  |
| description | TEXT    | 标签描述  |
| parent_id   | INTEGER | 父标签ID |

---

## 标签类型

| 类型      | 含义   |
| ------- | ---- |
| cuisine | 菜系   |
| taste   | 口味   |
| type    | 餐厅类型 |
| scene   | 场景   |
| feature | 特征   |

---

# 4.2 餐厅表（restaurant）

用于缓存餐厅基础信息。

---

## 表结构

```sql id="5l5v9o"
CREATE TABLE restaurant (
    id          TEXT PRIMARY KEY,
    name        TEXT NOT NULL,
    category    TEXT,
    address     TEXT,
    latitude    REAL,
    longitude   REAL,
    rating      REAL DEFAULT 0,
    avg_price   REAL DEFAULT 0,
    updated_at  TEXT
);
```

---

## 字段说明

| 字段         | 类型   | 说明       |
| ---------- | ---- | -------- |
| id         | TEXT | 高德POI ID |
| name       | TEXT | 餐厅名称     |
| category   | TEXT | 高德分类     |
| address    | TEXT | 地址       |
| latitude   | REAL | 纬度       |
| longitude  | REAL | 经度       |
| rating     | REAL | 评分       |
| avg_price  | REAL | 人均价格     |
| updated_at | TEXT | 更新时间     |

# 4.8 推荐记录表（recommendation）

用于保存推荐结果。

---

## 表结构

```sql id="vzr9wr"
CREATE TABLE recommendation (
    id              TEXT PRIMARY KEY,
    query_id        TEXT REFERENCES user_query(id),
    restaurant_id   TEXT,
    restaurant_name TEXT,
    rank            INTEGER,
    final_score     REAL,
    score_distance  REAL,
    score_price     REAL,
    score_rating    REAL,
    score_tag       REAL,
    matched_tags    TEXT,
    reason_hint     TEXT,
    explain_json    TEXT,
    created_at      TEXT
);
```

---

## 字段说明

| 字段             | 类型      | 说明       |
| -------------- | ------- | -------- |
| rank           | INTEGER | 推荐排名     |
| final_score    | REAL    | 综合得分     |
| score_distance | REAL    | 距离评分     |
| score_price    | REAL    | 价格评分     |
| score_rating   | REAL    | 评分权重     |
| score_tag      | REAL    | 标签匹配评分   |
| matched_tags   | TEXT    | 匹配标签JSON |
| reason_hint    | TEXT    | 推荐原因JSON |
| explain_json   | TEXT    | 完整推荐解释   |

---

# 4.9 反馈表（feedback）

用于保存用户反馈结果。

---

## 表结构

```sql id="8wtbls"
CREATE TABLE feedback (
    id                TEXT PRIMARY KEY,
    user_id           TEXT,

    rating            INTEGER,
    chosen            INTEGER DEFAULT 0,
    created_at        TEXT
);
```

---

## 字段说明

| 字段     | 类型      | 说明     |
| ------ | ------- | ------ |
| rating | INTEGER | 用户评分   |

* 查询用户历史记录

---

## 推荐记录索引

```sql id="3m6ytp"
CREATE INDEX idx_recommendation_query_id
ON recommendation(query_id);
```

用于：

* 查询某次推荐结果

---

## 行为记录索引

```sql id="ntkww5"
CREATE INDEX idx_interaction_user_id
ON interaction(user_id);
```

用于：

* 用户行为分析

---

## 用户偏好索引

```sql id="wy8x8g"
CREATE INDEX idx_user_tag_pref_user
ON user_tag_preference(user_id);
```

用于：

* 快速加载用户画像
