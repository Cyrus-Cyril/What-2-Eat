## 数据库设计目标

本系统数据库主要用于支持以下核心业务：

- 用户基础信息管理
- 用户偏好与画像存储
- 周边餐馆信息管理
- 查询记录与推荐记录保存
- 用户反馈闭环优化
- 历史行为分析与推荐优化

数据库设计遵循以下原则：

- 结构清晰，满足系统核心业务流程
- 支持推荐闭环与后续算法扩展
- 保证一定的查询性能与可维护性
- 支持后期个性化推荐与行为分析
- 适合课程项目开发规模与部署需求

本项目采用 MySQL 作为主要关系型数据库。

| 实体                  | 作用                   |

# 2. 数据库表设计

------

# （1）用户表 user

用于保存用户基础信息与个性偏好。

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    
    username VARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50),

    gender VARCHAR(10),
    age INT,

    preference_json JSON,

    budget_preference_min INT DEFAULT 0,
    budget_preference_max INT DEFAULT 100,

    distance_preference INT DEFAULT 3000,

    spicy_preference FLOAT DEFAULT 0.5,
    sweet_preference FLOAT DEFAULT 0.5,
    healthy_preference FLOAT DEFAULT 0.5,

    register_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_active_time DATETIME,

    status TINYINT DEFAULT 1
);
```

------

## 字段说明

| 字段                      | 含义         |
| ------------------------- | ------------ |
| preference_json           | 用户偏好标签 |
| budget_preference_min/max | 用户常用预算 |
| distance_preference       | 最大接受距离 |
| spicy_preference          | 辣度偏好     |
| healthy_preference        | 健康饮食倾向 |

------

# （2）餐馆表 restaurant

用于保存餐馆基础信息。

```sql
CREATE TABLE restaurant (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    external_id VARCHAR(100),

    name VARCHAR(100) NOT NULL,

    category VARCHAR(50),

    tags JSON,

    address VARCHAR(255),

    latitude DOUBLE,
    longitude DOUBLE,

    avg_price FLOAT,

    rating FLOAT,

    review_count INT DEFAULT 0,

    delivery_support TINYINT DEFAULT 0,

    business_status VARCHAR(20),

    source_platform VARCHAR(50),

    update_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_location(latitude, longitude),
    INDEX idx_category(category),
    INDEX idx_rating(rating)
);
```

------

## 字段说明

| 字段             | 含义           |
| ---------------- | -------------- |
| external_id      | 地图平台餐馆ID |
| tags             | 餐馆标签       |
| delivery_support | 是否支持外卖   |
| business_status  | 营业状态       |

------

# （3）查询记录表 query_record

用于记录用户每次查询请求。

```sql
CREATE TABLE query_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    user_id BIGINT NOT NULL,

    query_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    location_text VARCHAR(255),

    latitude DOUBLE,
    longitude DOUBLE,

    budget_min INT,
    budget_max INT,

    people_count INT,

    taste_tags JSON,

    avoid_tags JSON,

    max_distance INT,

    recommend_mode VARCHAR(30),

    FOREIGN KEY (user_id) REFERENCES user(id),

    INDEX idx_user(user_id),
    INDEX idx_time(query_time)
);
```

------

# （4）推荐记录表 recommendation_record

用于记录系统给出的推荐结果。

```sql
CREATE TABLE recommendation_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    query_id BIGINT NOT NULL,

    restaurant_id BIGINT NOT NULL,

    recommend_score FLOAT,

    rank_index INT,

    score_distance FLOAT,
    score_price FLOAT,
    score_rating FLOAT,
    score_preference FLOAT,

    recommend_reason TEXT,

    clicked TINYINT DEFAULT 0,

    chosen TINYINT DEFAULT 0,

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (query_id) REFERENCES query_record(id),

    FOREIGN KEY (restaurant_id) REFERENCES restaurant(id),

    INDEX idx_query(query_id),
    INDEX idx_restaurant(restaurant_id)
);
```

------

## 字段说明

| 字段             | 含义             |
| ---------------- | ---------------- |
| recommend_score  | 综合推荐分数     |
| score_distance   | 距离评分         |
| score_price      | 价格评分         |
| score_preference | 偏好匹配评分     |
| clicked          | 用户是否点击     |
| chosen           | 用户是否最终选择 |

------

# （5）反馈记录表 feedback_record

用于保存用户反馈。

```sql
CREATE TABLE feedback_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    recommendation_id BIGINT NOT NULL,

    user_id BIGINT NOT NULL,

    restaurant_id BIGINT NOT NULL,

    rating INT,

    would_revisit TINYINT DEFAULT 0,

    actual_consumption FLOAT,

    feedback_text TEXT,

    feedback_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (recommendation_id)
        REFERENCES recommendation_record(id),

    FOREIGN KEY (user_id)
        REFERENCES user(id),

    FOREIGN KEY (restaurant_id)
        REFERENCES restaurant(id),

    INDEX idx_user(user_id),
    INDEX idx_restaurant(restaurant_id)
);
```

------

# （6）收藏表 favorite_restaurant

用于保存用户收藏餐馆。

```sql
CREATE TABLE favorite_restaurant (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    user_id BIGINT NOT NULL,

    restaurant_id BIGINT NOT NULL,

    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES user(id),

    FOREIGN KEY (restaurant_id)
        REFERENCES restaurant(id),

    UNIQUE KEY uk_user_restaurant(user_id, restaurant_id)
);
```

------

# （7）用户行为日志表 user_behavior_log

用于记录用户行为数据。

```sql
CREATE TABLE user_behavior_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    user_id BIGINT,

    behavior_type VARCHAR(50),

    target_id BIGINT,

    target_type VARCHAR(50),

    behavior_time DATETIME DEFAULT CURRENT_TIMESTAMP,

    extra_data JSON,

    INDEX idx_user(user_id),
    INDEX idx_behavior(behavior_type)
);
```

| 表                    | 索引字段                |

# 5. 数据库优化设计

## （1）缓存机制

对于高频餐馆数据，可采用 Redis 缓存：

- 热门餐馆缓存
- 附近餐馆缓存
- 推荐结果缓存

减少重复调用地图API。

------

## （2）反规范化设计

在 recommendation_record 中保留：

- 餐馆名称
- 餐馆类别

减少高频联表查询。

------

## （3）日志化设计

系统保留完整：

- 查询记录
- 推荐记录
- 用户反馈

便于：

- 推荐算法优化
- 用户画像分析
- 系统调试
