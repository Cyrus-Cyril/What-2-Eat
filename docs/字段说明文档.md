## 1. 接口说明

### 1.1 调用方式

```python
from app.services.data_entry import get_candidate_restaurants

restaurants = get_candidate_restaurants(
    longitude: float,   # 经度
    latitude: float,    # 纬度
    radius: int = 1000,  # 搜索半径（米），默认1000
    max_count: int = 20 # 最大返回数量，默认20
) -> list[dict]
```

### 1.2 返回格式

```json
[
  {
    "restaurant_id": "...",
    "name": "...",
    ...
  },
  ...
]
```

## 3. 字段详细说明

### 3.1 restaurant_id
- **来源**：高德地图POI的`id`字段
- **用途**：作为餐馆的唯一标识，用于去重和关联其他数据
- **注意**：非自增ID，是第三方ID

### 3.2 name
- **来源**：高德地图POI的`name`字段
- **格式**：字符串，已去除首尾空格
- **最大长度**：255字符

### 3.3 category
- **来源**：高德地图POI的`type`字段，取分号分隔的最后一段
- **示例**：
  - 高德返回：`餐饮服务;中餐厅;川菜`
  - 取值：`川菜`
- **常见值**：中餐厅、小餐厅、面馆、火锅、烧烤、快餐、奶茶店等

### 3.4 distance_m
- **来源**：高德地图POI的`distance`字段
- **单位**：米（m）
- **说明**：高德计算的距搜索中心点的直线距离

### 3.5 rating
- **来源**：高德地图POI的`business.rating`字段
- **范围**：0.0 ~ 5.0
- **无数据时**：返回 0.0
- **精度**：保留1位小数

### 3.6 avg_price
- **来源**：高德地图POI的`business.cost`字段
- **单位**：元（CNY）
- **无数据时**：返回 0.0
- **精度**：保留2位小数

### 3.7 address
- **来源**：高德地图POI的`address`字段
- **处理**：若高德返回列表，取第一项
- **无数据时**：返回字符串 `"地址未知"`

### 3.8 latitude / longitude
- **来源**：高德地图POI的`location`字段，格式为 `"经度,纬度"`
- **坐标系**：GCJ-02（国测局坐标系，俗称"火星坐标系"）
- **注意**：与 GPS 原始坐标存在偏移，国内应用推荐使用此坐标系

## 4. 数据质量说明

### 4.1 清洗规则

以下情况的数据会被丢弃（不返回）：
- `name` 为空
- `id`（restaurant_id）为空
- `location` 格式异常（如非 `"经度,纬度"` 格式）
- 坐标解析失败

### 4.2 成功率

根据测试，正常坐标返回有效数据的成功率约为 **100%**（高德API稳定返回）

### 4.3 数据量

- 单次最大返回：**25条**（受高德API限制）
- 实际返回数量取决于搜索半径内餐馆密度

## 5. 数据库建表建议

```sql
CREATE TABLE restaurants (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '自增主键',
    restaurant_id VARCHAR(64) NOT NULL UNIQUE COMMENT '高德POI唯一ID',
    name VARCHAR(255) NOT NULL COMMENT '餐馆名称',
    category VARCHAR(64) NOT NULL COMMENT '餐馆类别',
    distance_m INT NOT NULL DEFAULT 0 COMMENT '距用户距离（米）',
    rating DECIMAL(2,1) NOT NULL DEFAULT 0.0 COMMENT '评分 0.0~5.0',
    avg_price DECIMAL(8,2) NOT NULL DEFAULT 0.0 COMMENT '人均消费（元）',
    address VARCHAR(512) NOT NULL COMMENT '详细地址',
    latitude DECIMAL(10,7) NOT NULL COMMENT '纬度',
    longitude DECIMAL(10,7) NOT NULL COMMENT '经度',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_category (category),
    INDEX idx_rating (rating),
    INDEX idx_distance (distance_m)
) COMMENT '候选餐馆表';
```
