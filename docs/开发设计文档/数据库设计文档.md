# 1. 文档概述

## 1.1 文档目的

本文档用于说明“今天吃什么”智能推荐系统的 MySQL 数据库结构设计。

数据库主要用于实现：

* 餐厅信息缓存
* 标签化推荐体系
* 用户偏好建模
* 推荐结果存储
* 用户行为分析
* 推荐反馈闭环
* 分布式推荐查询支持

系统数据库支持：

```text id="l6l7yr"
用户查询
→ 餐馆并行检索
→ 推荐排序
→ 用户反馈
→ 偏好更新
```

完整推荐链路。

# 4. 数据表设计

---

# 4.1 标签表（tag）

用于管理餐厅标签体系。

---

## 表结构

```sql id="xhh4b0"
DROP TABLE IF EXISTS tag;

CREATE TABLE tag (
    id INT PRIMARY KEY AUTO_INCREMENT,

    name VARCHAR(100) NOT NULL UNIQUE,

    type VARCHAR(50) NOT NULL,

    description TEXT,

    parent_id INT,

    CONSTRAINT fk_parent_tag
        FOREIGN KEY (parent_id)
        REFERENCES tag(id)
);
```

---

# （3）用户偏好表 user_tag_preference

| 字段          | 类型           | 说明    |
| ----------- | ------------ | ----- |
| id          | INT          | 标签ID  |
| name        | VARCHAR(100) | 标签名称  |
| type        | VARCHAR(50)  | 标签类型  |
| description | TEXT         | 标签描述  |
| parent_id   | INT          | 父标签ID |

---

## 标签类型

| 类型      | 含义   |
| ------- | ---- |
| cuisine | 菜系   |
| taste   | 口味   |
| type    | 餐厅类型 |
| scene   | 场景   |
| feature | 特征   |

# 4.2 餐厅表（restaurant）

用于缓存餐厅基础信息。

---

## 表结构

```sql id="1l5jlwm"
DROP TABLE IF EXISTS restaurant;

CREATE TABLE restaurant (
    id VARCHAR(100) PRIMARY KEY,

    name VARCHAR(100) NOT NULL,

    category VARCHAR(255),

    address VARCHAR(255),

    latitude DOUBLE,

    longitude DOUBLE,

    rating FLOAT DEFAULT 0,

    avg_price FLOAT DEFAULT 0,

    updated_at DATETIME
);
```

---

## 字段说明

| 字段         | 类型           | 说明       |
| ---------- | ------------ | -------- |
| id         | VARCHAR(100) | 高德POI ID |
| name       | VARCHAR(100) | 餐厅名称     |
| category   | VARCHAR(255) | 高德分类     |
| address    | VARCHAR(255) | 地址       |
| latitude   | DOUBLE       | 纬度       |
| longitude  | DOUBLE       | 经度       |
| rating     | FLOAT        | 餐厅评分     |
| avg_price  | FLOAT        | 人均价格     |
| updated_at | DATETIME     | 更新时间     |

---

# 4.3 餐厅标签关联表（restaurant_tag）

用于实现餐厅与标签的多对多关系。

---

## 表结构

```sql id="jlwm2m"
DROP TABLE IF EXISTS restaurant_tag;

CREATE TABLE restaurant_tag (
    restaurant_id VARCHAR(100) NOT NULL,

    tag_id INT NOT NULL,

    weight FLOAT DEFAULT 1.0,

    PRIMARY KEY (restaurant_id, tag_id),

    CONSTRAINT fk_restaurant_tag_restaurant
        FOREIGN KEY (restaurant_id)
        REFERENCES restaurant(id),

    CONSTRAINT fk_restaurant_tag_tag
        FOREIGN KEY (tag_id)
        REFERENCES tag(id)
);
```

# 4.5 用户长期偏好表（user_tag_preference）

用于保存用户长期标签偏好。

---

## 表结构

```sql id="jlwm4m"
DROP TABLE IF EXISTS user_tag_preference;

CREATE TABLE user_tag_preference (
    user_id VARCHAR(100) NOT NULL,

    tag_id INT NOT NULL,

    preference FLOAT DEFAULT 0.5,

    updated_at DATETIME,

    PRIMARY KEY (user_id, tag_id),

    CONSTRAINT fk_user_pref_user
        FOREIGN KEY (user_id)
        REFERENCES app_user(id),

    CONSTRAINT fk_user_pref_tag
        FOREIGN KEY (tag_id)
        REFERENCES tag(id)
);
```

---

```sql id="jlwm5m"

CREATE TABLE interaction (
```

# 4.8 推荐记录表（recommendation）

用于保存推荐结果。

---

## 表结构

```sql id="jlwm7m"
DROP TABLE IF EXISTS recommendation;

CREATE TABLE recommendation (
    id VARCHAR(100) PRIMARY KEY,

    query_id VARCHAR(100),

    restaurant_id VARCHAR(100),

    restaurant_name VARCHAR(100),

    rank_index INT,

    final_score FLOAT,

    score_distance FLOAT,

    score_price FLOAT,

    score_rating FLOAT,

    score_tag FLOAT,

    matched_tags JSON,

    reason_hint JSON,

    explain_json JSON,

    created_at DATETIME,

    CONSTRAINT fk_recommendation_query
        FOREIGN KEY (query_id)
        REFERENCES user_query(id)
);
```

---

# （7）推荐结果表 recommendation

| 字段             | 类型    | 说明     |
| -------------- | ----- | ------ |
| rank_index     | INT   | 推荐排名   |
| final_score    | FLOAT | 综合评分   |
| score_distance | FLOAT | 距离评分   |
| score_price    | FLOAT | 价格评分   |
| score_rating   | FLOAT | 评分权重   |
| score_tag      | FLOAT | 标签匹配评分 |
| matched_tags   | JSON  | 匹配标签   |
| reason_hint    | JSON  | 推荐原因   |
| explain_json   | JSON  | 完整推荐解释 |

# 4.9 用户反馈表（feedback）

用于保存用户反馈结果。

---

## 表结构

```sql id="jlwm8m"
DROP TABLE IF EXISTS feedback;

CREATE TABLE feedback (
    id VARCHAR(100) PRIMARY KEY,

    user_id VARCHAR(100),

    recommendation_id VARCHAR(100),

    restaurant_id VARCHAR(100),

    rating INT,

    chosen TINYINT DEFAULT 0,

    created_at DATETIME,

    CONSTRAINT fk_feedback_user
        FOREIGN KEY (user_id)
        REFERENCES app_user(id),

    CONSTRAINT fk_feedback_recommendation
        FOREIGN KEY (recommendation_id)
        REFERENCES recommendation(id)
);
```
